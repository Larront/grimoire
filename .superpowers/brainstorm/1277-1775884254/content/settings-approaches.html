A. Single Scroll — Stacked Cards

All three sections as stacked cards in a single scrollable column. Simple, scannable, nothing hidden. The old code used this pattern.

Pros

  • Everything visible at once
  • Simple to build and maintain
  • Familiar settings pattern

Cons

  • Could feel sparse with only 3 sections
  • Long page as sections grow

B. Tabbed Sections

Tab bar across the top — Integrations, Vault, Preferences. Each tab shows its own panel. Scales well if more integrations or preferences appear later.

Pros

  • Scales well with more content
  • Focused view per section
  • Feels polished and organized

Cons

  • More complex to build (tab state)
  • Hides content behind clicks
  • Overkill for 3 small sections

C. Side Navigation

A left-hand nav inside the settings page that jumps between sections. macOS System Settings style. Scrolls content on the right.

Pros

  • Professional, familiar pattern
  • Scales to many sections
  • Quick access to any section

Cons

  • A lot of chrome for 3 items
  • Nested sidebars (app sidebar + settings nav)
  • Most complex to build

My recommendation: A (Stacked Cards) — with only 3 compact sections, everything fits on one screen. Tabs and side-nav add navigation overhead that isn't justified yet. Easy to add more sections later by stacking more cards.
